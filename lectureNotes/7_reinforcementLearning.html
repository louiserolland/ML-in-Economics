Reinforcement Learning

Markov Process

Markov Reward Process

Return is the discounted sum of rewards (net present value): \[\begin{align*} G_t = \sum_{k=0}^{\infty} \gamma^k R_{t+k+1} \end{align*}\]

Markov Decision Process

Partially Observable MDP (POMDP)

Policy \(\pi(a_t|s_t)\)

Value Function: Expected Return

  • State-value function \(V^{\pi}(s_t)=\mathbb{E}[G_t|s_t,\pi]\)
  • State-action value function \(Q^{\pi}(s_t,a_t)=\mathbb{E}[G_t|s_t,a_t,\pi]\)
  • Optimal state-value funtion \(V^*(s_t)=\max_{\pi}V^{\pi}(s_t)\)
  • Optimal state-action value function \(Q^*(s_t,a_t)=\max_{\pi}Q^{\pi}(s_t,a_t)\)
  • Optimal Policy \(\pi^*(a_t|s_t)\leftarrow \arg\max_{a_t}Q^*(s_t,a_t)\)

Bellman Equation

\[\begin{align*} V^{\pi}(s_t) &= \sum_{a_t} \pi(a_t|s_t) Q(s_t,a_t) \end{align*}\]

\[\begin{align*} Q^{\pi}(s_t,a_t) &= r(s_t,a_t) + \gamma \sum_{s_{t+1}} P(s_{t+1}|s_t,a_t) V^{\pi}(s_{t+1}) \end{align*}\]

and therefore

\[\begin{align*} V^{\pi}(s_t) &= \sum_{a_t} \pi(a_t|s_t) \left[ r(s_t,a_t) + \gamma \sum_{s_{t+1}} P(s_{t+1}|s_t,a_t) V^{\pi}(s_{t+1}) \right]\\ Q^{\pi}(s_t,a_t) &= r(s_t,a_t) + \gamma \sum_{s_{t+1}} P(s_{t+1}|s_t,a_t) \sum_{a_{t+1}} \pi(a_{t+1}|s_{t+1}) Q^{\pi}(s_{t+1},a_{t+1}) \end{align*}\]

Tabular Reinforcement Learning

(Prince 2023, Ch. 19.3)

Tabular RL algorithms do not rely on function approximation.

Model-Based (MB) vs. Model-Free (MF)

  • Model-based: \(P(s_{t+1}|s_t,a_t)\) and \(r(s_t,a_t)\) are known
    • Dynamic programming
  • Model-free: no model of \(P(s_{t+1}|s_t,a_t)\) and/or \(r(s_t,a_t)\)
    • Value estimation: estimate \(Q^*(s_t,a_t)\) and derive \(\pi^*(a_t|s_t)\)
      • Monte Carlo value estimation
      • Temporal difference (TD) methods
    • Policy estimation: estimate \(\pi^*(a_t|s_t)\) directly (using gradient descent)

Dynamic Programming (MB)

\[\begin{align*} V(s_t) &\leftarrow \sum_{a_t} \pi(a_t|s_t) \left[ r(s_t,a_t) + \gamma \sum_{s_{t+1}} P(s_{t+1}|s_t,a_t) V(s_{t+1}) \right] \\ \pi(a_t|s_t) &\leftarrow \arg\max_{a_t} \left[ r(s_t,a_t) + \gamma \sum_{s_{t+1}} P(s_{t+1}|s_t,a_t) V(s_{t+1}) \right] \end{align*}\]

Policy iteration

initialize π(s), V(s)
policy_stable ← false
while not policy_stable:
  do:
    Δ ← 0
    for each state s:
      v ← V(s)
      V(s) ← Σ_a π(a|s)[r(s,a) + γΣ_{s'}P(s'|s,a)V(s')]
      Δ ← max(Δ, |v - V(s)|)
  while Δ ≥ θ
  policy_stable ← true
  for each state s:
    a ← π(s)
    π(s) ← argmax_a [r(s,a) + γΣ_{s'}P(s'|s,a)V(s')]
    if a ≠ π(s): policy_stable ← false

Evaluate current \(\pi\) fully before updating \(\pi\); until it no longer changes.

Value iteration

initialize V(s)
do:
  Δ ← 0
  for each state s:
    v ← V(s)
    V(s) ← max_a [r(s,a) + γΣ_{s'}P(s'|s,a)V(s')]
    Δ ← max(Δ, |v - V(s)|)
while Δ ≥ θ
for each state s:
  π(s) ← argmax_a [r(s,a) + γΣ_{s'}P(s'|s,a)V(s')]

Update \(V\) with Bellman optimality backups until convergence.

Monte Carlo Value Estimation (MF)

On-policy: Alternate between estimating \(Q(s,a) \approxeq \frac{1}{N}\sum_e G_t(s,a,\pi)\) from a series of episodes \(e\), and updating policy according to \[\begin{align*} \pi(a|s) \leftarrow \arg\max_a Q(s,a) \end{align*}\]

Off-policy: Target policy \(\pi\) learned with episodes from behavior policy \(\pi'\).

Temporal Difference (TD) Learning (MF)

Combine bootstrapping of dynamic programming with sampling of MC, updating values & policy while agent interacts with environment.

SARSA (on-policy): \[\begin{align*} Q(s_t,a_t) &\leftarrow Q(s_t,a_t) + \alpha \left[ r(s_t,a_t) + \gamma Q(s_{t+1},a_{t+1}) - Q(s_t,a_t) \right] \end{align*}\]

Q-learning (off-policy): \[\begin{align*} Q(s_t,a_t) &\leftarrow Q(s_t,a_t) + \alpha \left[ r(s_t,a_t) + \gamma \max_{a} Q(s_{t+1},a) - Q(s_t,a_t) \right] \end{align*}\]

Updates are contraction mappings so \(Q(s,a)\) converges.

Deep Q-Learning (MF)

(Prince 2023, Ch. 19.4)

Fitted Q-Learning

ML model \(q(\mathbf{s}_t,a;\phi)\) where \(\mathbf{s}_t\) is now a state vector.

\[\begin{align*} \phi \leftarrow \phi + \alpha \left[ r(s_t,a_t) + \gamma \max_{a} q(s_{t+1},a;\phi) - q(s_t,a_t;\phi) \right] \nabla_{\phi} q(s_t,a_t;\phi) \end{align*}\] to minimize the least squares loss \[\begin{align*} L(\phi) = \left( r(s_t,a_t) + \gamma \max_{a} q(s_{t+1},a;\phi) - q(s_t,a_t;\phi) \right)^2 \end{align*}\]

Semi-gradient: the update uses \(\nabla_\phi L\) only through \(q(s_t,a_t;\phi)\); the bootstrap \(r + \gamma \max_{a} q(s_{t+1},a;\phi)\) is treated as fixed w.r.t. \(\phi\) in this step.

Convergence no longer guaranteed.

Deep Q-Network (DQN)

  • Experience replay: batch sampled from replay buffer of recent \((s_t,a_t,r_t,s_{t+1})\) tuples.
  • \(\max_{a} q(s_{t+1},a;\phi^{-})\) with \(\phi^{-}\) updated only periodically

Policy Gradient Methods (MF)

(Prince 2023, Ch. 19.5 and Ch. 19.6)

Gradient Update

Directly learn policy \(\pi(a_t|\mathbf{s}_t;\mathbf{\theta})\):

\[\begin{align*} \arg\max_{\mathbf{\theta}} \int \left(\sum_{t=1}^T r(\mathbf{s}_t,a_t)\right) P(\mathbf{s}_1)\prod_{t=1}^T \pi(a_t|\mathbf{s}_t;\mathbf{\theta})P(\mathbf{s}_{t+1}|\mathbf{s}_t,a_t) d\mathbf{s}_1da_1 \ldots d\mathbf{s}_T d a_T \end{align*}\]

yields (approximating integral with sum over \(I\) episodes with trajectories \(\mathbf{\tau}_i\)) the gradient ascent update

\[\begin{align*} \mathbf{\theta} &\leftarrow \mathbf{\theta} + \frac{\alpha }{I}\sum_{i=1}^I\frac{1}{P(\mathbf{\tau}_i|\mathbf{\theta})}\frac{\partial P(\mathbf{\tau}_i|\mathbf{\theta})}{\partial \mathbf{\theta}} \sum_{t=1}^T r_{it} \end{align*}\]

which updates \(\mathbf{\theta}\) to increase likelihood of observed trajectories in proportion to their returns.

Gradient Update

Equivalently \[\begin{align*} \mathbf{\theta} &\leftarrow \mathbf{\theta} + \frac{\alpha }{I}\sum_{i=1}^I\frac{ \partial \log P(\mathbf{\tau}_i|\mathbf{\theta})}{\partial \mathbf{\theta}} \sum_{t=1}^T r_{it} \end{align*}\] which (writing out \(P(\tau_i|\theta)\) and simplifying) gives rise to \[\begin{align*} \mathbf{\theta} &\leftarrow \mathbf{\theta} + \frac{\alpha }{I}\sum_{i=1}^I\sum_{t=1}^T \nabla_{\mathbf{\theta}} \log \pi(a_{it}|\mathbf{s}_{it};\mathbf{\theta}) \sum_{k=t}^T r_{ik} \end{align*}\]

REINFORCE algorithm

MC policy gradient method: sample episodes from \(\pi_{a}(s;\theta)\) (e.g. neural network with softmax output).

\[\begin{align*} G_{it} &= \sum_{k=t+1}^T \gamma^{k-t-1} r_{ik} \\ \theta &\leftarrow \theta + \alpha \gamma^t \nabla_{\theta}\log \pi_{a_{it}}(s_{it};\mathbf{\theta}) G_{it} \end{align*}\]

initialize policy parameters \mathbf{\theta}
repeat:
  sample I episodes \tau_i \sim \pi_{\mathbf{\theta}}(\cdot|\mathbf{s})
  \Delta \mathbf{\theta} \leftarrow \mathbf{0}
  for i = 1,\ldots,I:
    G_{i,T_i+1} \leftarrow 0
    for t = T_i,\ldots,1:
      G_{it} \leftarrow r_{it} + \gamma G_{i,t+1}
    for t = 1,\ldots,T_i:
      \Delta \mathbf{\theta} \leftarrow \Delta \mathbf{\theta} + \nabla_{\mathbf{\theta}} \log \pi(a_{it}|\mathbf{s}_{it};\mathbf{\theta})\, G_{it}
  \mathbf{\theta} \leftarrow \mathbf{\theta} + \frac{\alpha}{I}\Delta \mathbf{\theta}
until convergence

Baselines

In practice, subtracting a baseline \(b(\mathbf{s}_{it})\) gives \[\begin{align*} \mathbf{\theta} \leftarrow \mathbf{\theta} + \frac{\alpha}{I}\sum_{i=1}^{I}\sum_{t=1}^{T}\nabla_{\mathbf{\theta}}\log \pi(a_{it}|\mathbf{s}_{it};\mathbf{\theta})\left(G_{it}-b(\mathbf{s}_{it})\right) \end{align*}\] which keeps the gradient unbiased and reduces variance.

Actor-Critic Methods

Temporal difference (TD) policy gradient method.

\[\begin{align*} \theta &\leftarrow \theta + \frac{\alpha }{I}\sum_{i=1}^I\sum_{t=1}^T \nabla_{\theta} \log \pi(a_{it}|\mathbf{s}_{it};\theta) \left(r_{it} + \gamma v(\mathbf{s}_{i,t+1},\phi) - v(\mathbf{s}_{i,t},\phi) \right)\\ L(\phi) &= \sum_{i=1}^I\sum_{t=1}^T \left( r_{it} + \gamma v(\mathbf{s}_{i,t+1},\phi) - v(\mathbf{s}_{i,t},\phi) \right)^2 \end{align*}\] where \(\pi(\mathbf{s}_t,\mathbf{\theta})\) is the actor and \(v(\mathbf{s}_t,\phi)\) the critic.

Trust Region Policy Optimization (TRPO)

Constrain policy updates by a KL trust region to avoid large unstable policy jumps.

\[\begin{align*} \max_{\theta}\; \mathbb{E}_t\!\left[\frac{\pi_\theta(a_t|s_t)}{\pi_{\theta_{\mathrm{old}}}(a_t|s_t)}\hat A_t\right] \quad \text{s.t.}\quad \mathbb{E}_t\!\left[D_{\mathrm{KL}}\!\left(\pi_{\theta_{\mathrm{old}}}(\cdot|s_t)\,\|\,\pi_{\theta}(\cdot|s_t)\right)\right]\le \delta \end{align*}\]

Proximal Policy Optimization (PPO)

Actor-critic method that limits policy changes per update using a clipped objective.

\[\begin{align*} r_t(\theta) &= \frac{\pi_\theta(a_t|s_t)}{\pi_{\theta_{\mathrm{old}}}(a_t|s_t)} \\ L^{\mathrm{CLIP}}(\theta) &= \mathbb{E}_t \left[\min \left(r_t(\theta)\hat A_t,\ \mathrm{clip}(r_t(\theta),1-\epsilon,1+\epsilon)\hat A_t\right)\right] \end{align*}\]

Typical loop:

collect rollouts with π_old
estimate advantages Â_t (with critic)
for k epochs:
  update actor with clipped objective
  update critic with value loss
set π_old ← π

Soft Actor-Critic (SAC)

Off-policy maximum-entropy actor-critic for continuous control. \[\begin{align*} J(\pi) = \mathbb{E}\left[\sum_t r(s_t,a_t) + \alpha \,\mathcal{H}\!\left(\pi(\cdot|s_t)\right)\right] \end{align*}\] The entropy term encourages exploration and makes training more stable.

Group Relative Policy Optimization (GRPO)

Policy optimization without a learned value critic, using relative rewards inside a sampled group.

\[\begin{align*} \hat A_i &= \frac{r_i - \bar r}{\sigma_r + \epsilon} \\ L^{\mathrm{GRPO}}(\theta) &= \mathbb{E}\left[\min\left(r_i(\theta)\hat A_i,\ \mathrm{clip}(r_i(\theta),1-\epsilon,1+\epsilon)\hat A_i\right)\right] - \beta\, D_{\mathrm{KL}}(\pi_\theta\|\pi_{\mathrm{ref}}) \end{align*}\]

Typical loop:

sample G outputs per prompt from π_old
compute reward r_i for each output
normalize rewards within group to get Â_i
update policy with clipped objective + KL penalty
set π_old ← π

Offline RL

(Prince 2023, Ch. 19.7)

Offline RL: Decision Transformer

Replace reward \(r_t\) with returns-to-go \(R_{t:T}=\sum_{t'=t}^T r_{t'}\) during training.

Reinforcement Learning from Human Feedback (RLHF)

From Policy Gradients to RLHF

RLHF can be viewed as policy gradient optimization on sequence models: \[\begin{align*} \max_{\pi}\; \mathbb{E}_{y\sim \pi(\cdot|x)}\left[r_{\text{RM}}(x,y)\right] - \beta\,D_{\mathrm{KL}}(\pi\|\pi_{\mathrm{ref}}) \end{align*}\] where \(r_{\text{RM}}\) is a learned reward model from human preferences.

Connection to stochastic choice and log utility: \[\begin{align*} P(y^+ \succ y^-|x) = \sigma\!\left(u(x,y^+) - u(x,y^-)\right) \end{align*}\] which is the Bradley-Terry / logistic random utility form, with latent utility \(u(x,y)\).

Preference learning step: \[\begin{align*} \max_{\psi}\sum_{(x,y^+,y^-)} \log \sigma\!\left(r_{\psi}(x,y^+) - r_{\psi}(x,y^-)\right) \end{align*}\] Then optimize the policy with PPO style updates (or critic-free variants such as GRPO).
